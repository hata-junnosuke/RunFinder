# Git ルール

## ブランチ命名規則

```
main              # 本番環境（保護ブランチ）
├── feature/*     # 新機能開発
├── fix/*         # バグ修正
├── refactor/*    # リファクタリング
```

この3種類のみ使用。`chore/*` や `docs/*` は使わない。

## git add の禁止事項

**`git add` はユーザーの明示的な承認なしに絶対に実行しない。**
必ず `git status` と `git diff` で変更内容を提示し、承認を得てから実行する。

## 実装後のチェックフロー

1. リント・テストを実行して通ることを確認
2. `/rev` でセルフレビュー
3. 「作業完了しました」と報告
4. ユーザーの承認を得てから `git add` を実行
