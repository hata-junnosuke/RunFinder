# RunFinder 要件定義書

## 1. プロダクト概要

現在地から指定した距離のランニングコースを動的に生成し、地図上に表示するWebアプリケーション。
ユーザー登録不要で、すぐにコースを生成できるシンプルな体験を提供する。

## 2. ターゲットユーザー

- ランニングをする人で、新しいコースを見つけたい人
- 旅行先や出張先など、土地勘のない場所で走りたい人
- 距離を決めて走りたいが、コース設計が面倒な人

## 3. 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Next.js (App Router) |
| バックエンドAPI | Hono on Bun |
| ORM | Drizzle ORM |
| DB | PostgreSQL |
| 地図 | Google Maps API (Maps JavaScript API + Directions API) |
| インフラ | Docker Compose (開発) / Vercel (デプロイ) |

## 4. フェーズ計画

### Phase 1: MVP（本ドキュメントのスコープ）
- 現在地取得
- 距離指定（スライダー、1km刻み）
- 周回コースを動的生成
- 地図上にルート表示

### Phase 2: コース保存
- 生成したコースをDBに保存
- 保存コースの一覧・詳細表示
- コースの削除

### Phase 3: 拡張機能（未定）
- 片道コース対応
- 方角指定、ペース指定
- コース共有（URL共有）
- お気に入り機能

## 5. Phase 1 機能要件

### F-001: 現在地の取得

| 項目 | 内容 |
|---|---|
| 概要 | ブラウザのGeolocation APIで現在地を取得する |
| トリガー | ページ読み込み時に位置情報の許可を求める |
| 正常系 | 緯度・経度を取得し、地図の中心に表示する |
| 異常系 | 許可拒否 → デフォルト位置（東京駅）を表示 |
| 備考 | 地図上のクリック/タップでも出発地点を変更できる |

### F-002: 距離の指定

| 項目 | 内容 |
|---|---|
| 概要 | スライダーUIで走りたい距離を指定する |
| 範囲 | 1km 〜 30km（1km刻み） |
| デフォルト | 5km |
| 表示 | スライダー横に現在の距離(km)を数値表示 |

### F-003: コースの生成

| 項目 | 内容 |
|---|---|
| 概要 | 現在地と距離を元に周回ランニングコースを生成する |
| トリガー | 「コースを生成」ボタン押下 |
| 生成方法 | Google Directions API を使用（詳細は設計書参照） |
| 再生成 | 「別のコースを生成」ボタンで別ルートを再生成できる |

### F-004: コースの地図表示

| 項目 | 内容 |
|---|---|
| 概要 | 生成されたコースを地図上にポリラインで表示する |
| 表示要素 | ルートライン（色付きポリライン）、出発地点マーカー |
| 情報表示 | 総距離(km) |
| 地図操作 | ズーム、パン |

## 6. 画面構成

### S-001: メイン画面（1画面構成）

#### モバイル（基準: 375px）

```
┌───────────────────────┐
│ RunFinder          📍 │
├───────────────────────┤
│                       │
│     Google Map        │
│   （画面の60%）        │
│                       │
├───────────────────────┤
│                       │
│ 距離: ─●───── 5 km    │
│                       │
│ [  コースを生成  ]     │
│                       │
├───────────────────────┤
│ 📏 5.2km              │
│ [別のコースを生成]     │
└───────────────────────┘
```

#### デスクトップ（768px〜）

```
┌──────────────────────────────────────────────┐
│ RunFinder                                 📍 │
├─────────────────────────┬────────────────────┤
│                         │                    │
│                         │ 距離: ─●── 5 km    │
│     Google Map          │                    │
│   （左側メイン）         │ [コースを生成]      │
│                         │                    │
│                         │ 📏 5.2km           │
│                         │ [別のコースを生成]   │
└─────────────────────────┴────────────────────┘
```

#### レイアウト方針

- **モバイルファースト**: スマホで使うことを前提に設計する
- モバイル: 地図が上部60%、操作パネルが下部（縦積み）
- デスクトップ: 地図が左、操作パネルが右（横並び）
- ブレイクポイント: 768px
- タッチ操作を考慮: ボタン・スライダーは最低44pxのタップ領域
- 片手操作を意識: 主要操作は画面下部に配置

## 7. API設計

### POST /api/courses/generate

コースを動的に生成する。

**リクエスト:**
```json
{
  "lat": 35.6812,
  "lng": 139.7671,
  "distance_km": 5
}
```

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| lat | number | ○ | 出発地点の緯度 |
| lng | number | ○ | 出発地点の経度 |
| distance_km | number | ○ | 希望距離（1〜30km） |

**レスポンス（成功）:**
```json
{
  "success": true,
  "data": {
    "route": {
      "overview_polyline": "encoded_polyline_string",
      "start": { "lat": 35.6812, "lng": 139.7671 }
    },
    "distance_km": 5.2
  }
}
```

**レスポンス（エラー）:**
```json
{
  "success": false,
  "error": "ルートを生成できませんでした。別の距離で試してください。"
}
```

### GET /api/health

ヘルスチェック。

**レスポンス:**
```json
{ "status": "ok" }
```
